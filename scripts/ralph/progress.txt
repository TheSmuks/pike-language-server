## Codebase Patterns
- WorkspaceIndex.upsertFile() builds symbol tables via buildSymbolTable(), which calls wireInheritance() internally
- wireInheritance() resolves same-file class inheritance by finding matching class declarations in the parent file scope
- Completion uses getCompletions() which takes a table + tree + CompletionContext (index, stdlibIndex, predefBuiltins, uri)
- Cross-file resolution in typeResolver uses resolveCrossFileType() which walks inherit/import declarations in the current file
- accessResolver.resolveAccessCore() resolves arrow/dot access: finds LHS declaration, calls resolveMemberAccess(), then findDeclUri()
- Tests use createTestServer() for LSP protocol tests and WorkspaceIndex directly for unit-level tests
- Corpus files are in corpus/files/ with naming convention: cross-inherit-*.pike, cross-import-*.pike
- The Declaration interface has optional sourceUri field for synthetic declarations from cross-file inheritance
- buildSymbolTable() accepts optional BuildOptions with index for cross-file inheritance wiring
- Declaration.assignedType stores initializer callee name for variables with declaredType absent or 'mixed'
- local_declaration (function-body) is NOT wrapped in declaration node; variable_decl (top-level) IS wrapped
- postfix_expr with childCount > 1 = call expression; childCount == 1 = simple reference
- declaredType || assignedType is wrong when declaredType is 'mixed'; must check !PRIMITIVE_TYPES.has(declaredType) first
- PRIMITIVE_TYPES is duplicated in symbolTable.ts, typeResolver.ts, and completion.ts — needs centralization
