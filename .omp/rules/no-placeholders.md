# No Placeholders Rule

## Purpose

Prevents agents from leaving `<!-- -->` or `<!-- e.g.` style HTML comment placeholders in markdown files. These indicate incomplete documentation and confuse adopting projects about what's real vs. template.

## Trigger

This rule activates via TTSR (zero upfront context cost) when the agent's output stream matches the trigger pattern.

## Trigger Pattern

Regex: `<!--\s*(e\.g\.|TODO|FIXME|replace me|fill in|<[^>]+>)`

### Why This Pattern?

| Pattern | What It Catches |
|---------|-----------------|
 `<!-- e.g.` | Common template indicator ("for example...") |
 `<!-- TODO` | Incomplete sections left for later |
 `<!-- FIXME` | Known issues not yet addressed |
 `<!-- replace me` | Direct "fill this in" instructions |
 `<!-- fill in` | Same as above |
 `<!-- <...>` | Generic placeholder comments with content between brackets |

## Scope

```
tool:write(**/*.md)
tool:edit(**/*.md)
```

This ensures the rule only applies to markdown file operations, not to all agent output.

## Example Triggers

### ❌ Triggers (should be caught)
```markdown
<!-- Describe the API endpoint here -->
<!-- e.g., GET /api/users/:id -->
<!-- TODO: Add error handling -->
```

### ✅ Does Not Trigger (legitimate comments)
```markdown
<!-- This file is generated by build script -->
<!-- See docs/api.md for details -->
```

## What To Do Instead

When you encounter a placeholder:
1. **Fill it in** with actual content if you have the information
2. **Remove it** if the section genuinely doesn't apply
3. **Ask** the user if the content requires domain knowledge you don't have

## Adapting This Rule

For adopting projects, you may customize the trigger pattern to catch project-specific placeholder formats:

```yaml
trigger: "PROJECT_NAME_HERE|___FILL_IN___|\\[TEMPLATE\\]"
```

Where:
- `PROJECT_NAME_HERE` — placeholder for the project name
- `___FILL_IN___` — legacy or custom placeholder format
- `\[TEMPLATE\]` — bracketed template markers

## Implementation Notes

This is a **TTSR rule** — it only activates when its pattern appears in the agent's output stream. This means:
- Zero upfront context cost (not loaded until needed)
- No false positives from unrelated markdown files
- Immediate detection as the agent writes problematic content
